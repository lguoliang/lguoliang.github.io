# Git 常用

# 提交规范

git-cz (opens new window)

<type>(<scope>): <subject>
  • type用于说明 commit 的类别，使用下列标识
feat:     新功能
fix:      修复
docs:     文档变更
style:    代码格式(不影响代码运行的变动)
refactor: 重构(既不是增加feature，也不是修复bug)
perf:     性能优化
test:     增加测试
chore:    构建过程或辅助工具的变动
ci:       CI 相关的变化
revert:   回退
build:    打包
  • scope 用来说明本次Commit影响的范围，即简要说明修改会涉及的部分,比如数据层、控制层、视图层等
  • subject 这次提交的简短描述

# 随机生成commit message

git commit -m"`curl -s http://whatthecommit.com/index.txt`"

# 强制覆盖本地代码：

// 本地代码重置为线上master分支(会覆盖本地所有与远程仓库上同名文件)
git reset --hard origin/master

# fetch

git remote add upstream ssh://git@120.78.197.207:7999/yqp/yqp-sa.git

git fetch upstream

git merge upstream/master

# Git 重新关联远程仓库地址的三种方法

1.修改命令

#查看关联的远程仓库
git remote -v

git remote set-url origin [url] 

例：git remote set-url origin XXX.git

2.先删后加

git remote rm origin 
git remote add origin [url]

例：git remote add origin XXX.git

3.直接修改.git文件夹下的config文件

[core]
    repositoryformatversion = 0
    filemode = false
    bare = false
    logallrefupdates = true
    symlinks = false
    ignorecase = true
[submodule]
    active = .
[remote "origin"]
    url = XXX.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[branch "master"]
    remote = origin
    merge = refs/heads/master
